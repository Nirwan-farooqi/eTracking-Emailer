1. System Overview

The eTracking Email Portal is a web-based tool for sending bulk, personalised HTML emails to eTracking customers. You upload a CSV file exported from the team records sheet, preview the generated emails, and send them ‚Äî all from a browser.

Key capabilities:

  • Multi-vehicle grouping ‚Äî customers with multiple vehicles receive one email listing all their vehicles, not one per row.
  • Template-driven ‚Äî each CSV row specifies which email template to use (renewal reminder, new account, device transfer, etc.).
  • Live progress ‚Äî a progress bar updates in real-time as each email is sent; you don't wait until all are done.
  • Payment image embedded ‚Äî the payment QR / options image is automatically embedded inline in renewal emails.
  • Markdown notes ‚Äî the notes column supports Markdown formatting.

2. Before You Start ‚Äî CSV Preparation

Export the CSV

Download the CSV from the team records sheet. Make sure it includes all required columns (see CSV Column Reference).

Fill in the email-template column

Every row must have a value in the email-template column. Valid values:

ValueWhen to use
renewal-pendingCustomer's renewal payment is due ‚Äî includes payment methods and QR image
renewal-doneCustomer has paid ‚Äî sends renewal confirmation
new-accountBrand new customer ‚Äî sends welcome + account setup details
device-transferDevice moved to a different vehicle
device-redoDevice reinstalled / redone
device-additionNew device added to existing account
‚ö† Important If any row has an empty or invalid email-template value, the upload will fail with an error. Fix the CSV and re-upload.

Check the Send-Email-To column

This must be a valid email address. Rows with a missing or invalid email address will be skipped with a FAILED status in the results.

Empty rows at the bottom

Excel often adds hundreds of blank rows at the end of CSV exports. These are automatically ignored ‚Äî the record count shown after upload will reflect only real data rows.

Templates are .hbs (Handlebars) files stored in the templates/ folder on the server. Each template receives the full customer data including the list of all vehicles.

Mixed templates in one account

If a customer's vehicles have different template values, the system picks the highest-priority one:

  1. renewal-done
  2. device-transfer / device-addition / device-redo
  3. new-account
  4. renewal-pending

6. Multi-Vehicle Accounts

A customer with multiple vehicles will appear as multiple rows in the CSV, all sharing the same ETC-number.

‚úÖ What the system does All rows with the same ETC number are automatically merged into one customer object. Only one email is sent listing every vehicle in a table. The total payable amount is the sum of all vehicle amounts. The expiry date shown is the earliest expiry across all vehicles.

Example: ETC-0189 (Ifran Iqbal) has 5 rows in the CSV. The portal shows 1 customer record, sends 1 email, and that email contains a table of all 5 vehicles.

You can verify this by uploading the CSV and checking the Records count ‚Äî it shows unique customers, not raw rows.

7. Previewing Emails

After uploading a CSV, you can preview any customer's email before sending:

  1. Enter the customer record number (1 = first customer, 2 = second, etc.) in the Record field.
  2. Click üîç Preview Email.
  3. The rendered email appears in an iframe below, exactly as the customer will receive it ‚Äî including the payment QR image.
‚Ñπ Payment image in preview The renewal-pending template embeds the payment QR image using a standard email technique (CID attachment). In the browser preview it is shown as a data URL so it renders correctly. In the actual email, it is a proper inline image attachment.

To check a customer after previewing: look at the preview header which shows the customer email, template name, and which record out of the total you are viewing.

8. Live Progress During Send

When you click üöÄ Send Emails, the portal does not wait for all emails to finish before showing results. Instead it uses a real-time stream:

  • A progress bar fills as each email is processed.
  • The label shows: Sending... 12 / 56   ‚úÖ 12   ‚ùå 0
  • Each email's result row appears in the table immediately after it is sent.
  • When all done, the bar turns full and shows: Done! ‚úÖ Sent: 56 ‚ùå Failed: 0 üìä Total: 56
‚ö† Do not close the browser tab While sending is in progress, keep the browser tab open. Closing it will disconnect the live stream (emails that have already been sent will not be resent, but you will lose visibility of remaining progress).

10. Email Log File

Every send attempt (success, failure, or dry-run) is recorded in logs/email-log.txt on the server:

2026-02-26T10:00:00.000Z | SUCCESS | ETC: 0055 | John Doe | john@example.com | Message ID: <abc123>
2026-02-26T10:00:02.000Z | FAILED  | ETC: 0221 | Jane Doe | jane@example.com | Connection timeout
2026-02-26T10:00:04.000Z | DRY_RUN | ETC: 0189 | Iqbal    | iqbal@ex.com     | Email prepared but not sent

This log is append-only. It grows over time and can be reviewed to audit when and to whom emails were sent.

11. Advanced: CLI Usage

The CLI processes all CSV files in the customers/ folder and supports powerful filtering options.

# Preview only ‚Äî no emails sent
node src/index.js --dry-run

# Send all customers in customers/ folder
node src/index.js

# Send to specific ETC numbers only
node src/index.js --etc 0055 0221 0189

# Only customers expiring within 7 days
node src/index.js --expiry 7

# Filter by package keyword
node src/index.js --package Platinum

# Limit to first 10 customers
node src/index.js --limit 10

# Skip the "are you sure?" confirmation
node src/index.js --yes

# Custom CSV folder
node src/index.js --folder ./exports/march

# Slower sending (5 seconds between emails)
node src/index.js --delay 5000

Processed files

After a successful CLI run, CSV files are moved from customers/ to processed/ with a timestamp prefix. Running again with the same file will skip it (unless the file content has changed).

‚úÖ Tip ‚Äî dry-run first Always run with --dry-run first to see a summary of who will be emailed before actually sending. Dry-run output HTML files are saved in output/ for manual inspection.

12. Troubleshooting

ProblemCauseFix
‚ùå Not Configured in header Missing .env credentials Ask system admin to set EMAIL_USER, EMAIL_PASS, FROM_NAME, FROM_EMAIL in .env
Upload fails with "email template" error A row has empty or invalid email-template value Open the CSV, find blank cells in the email-template column, fill them with a valid value (e.g. renewal-pending)
